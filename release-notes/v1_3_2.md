# v1.3.2 Command Introspection and Jira Token Handling

## New Features
- Added `get` command support for inspecting effective command inputs:
  - `gsloth get <command> prompt` prints the combined system prompt for `ask`, `review`, `pr`, `chat`, or `code`
  - `gsloth get <review|pr> <content|requirements> <id>` prints the wrapped provider payload that would be sent to the model

## Improvements
- Jira integration now supports `JIRA_FULL_BASE64_TOKEN` for environments that already provide a complete Base64-encoded Basic auth token.
- Jira REST API calls continue to support `JIRA_USERNAME`, `JIRA_API_PAT_TOKEN`, and `JIRA_CLOUD_ID` when a full token is not supplied.
