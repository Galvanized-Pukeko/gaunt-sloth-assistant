# v1.1.10 Custom Tool Validation Override

## New Features
- **Custom Tool Validation Override:** Custom tools can now selectively bypass security validation checks on a per-parameter basis. When validation fails, users are prompted to allow execution one-time or add permanent `allow` configuration. Each parameter accepts an `allow` array specifying which checks to skip: `absolute-paths`, `directory-traversal`, `shell-injection`, or `null-bytes`. This enables legitimate use cases like device paths (`/dev/ttyUSB0`) while maintaining security for other parameters.

## Bug Fixes
- **Validation Prompt Input:** Fixed character duplication when typing responses to validation override prompts. The readline interface now correctly handles terminal echo without duplicating input characters.
