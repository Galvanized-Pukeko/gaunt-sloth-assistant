# v1.5.0 Frontend-Fulfilled Tools and TypeScript 6

## New Features
- **Frontend-fulfilled tools (AG-UI):** Tools tagged with `metadata.client === true` are wrapped so their body calls `interrupt()` from `@langchain/langgraph`, suspending the agent on the same thread. The `/agents/:agentId/run` endpoint accepts `forwardedProps.command.resume` and routes resumed runs through the new `streamWithEventsResume` method, leaving the original tool object untouched.
- **Binary envelope round-trip:** AG-UI `express.json` body limit raised to 5mb so base64 payloads can travel through `forwardedProps`.

## Improvements
- **Streamed tool calls preserved across chunks:** Core now concatenates `AIMessageChunk`s across a round so `collapseToolCallChunks` sees the full `args` JSON instead of a single chunk's slice. The aggregate is reset after each `ToolMessage` flush so OpenAI's index-0 restart in the next round does not collide with previous groups. `invalid_tool_calls` are surfaced to the client instead of being dropped.
- `streamWithEvents` event processing extracted into a shared `processEventStream` used by both stream variants.

## Potentially Breaking Changes
- **TypeScript 6.0** is now used to build the packages. Per-package `tsconfig.json` files no longer set the deprecated `baseUrl` (paths remain valid because they are already written relative to each tsconfig). Downstream consumers compiling against the workspace types should ensure their toolchain accepts TS 6 output.

## Maintenance
- Routine dependency updates across `@langchain/*`, `langchain`, `langsmith`, `openai`, `@typescript-eslint/*`, and others.
- `@gaunt-sloth/tools` is now a workspace-linked package.
- Root workspace `package.json` is annotated and pinned to version `0` to make clear it is not published.
