# v1.4.0 Binary Model Output Materialization

## New Features
- **Binary output handling:** Models that return inline binary content (e.g. Gemini image generation) now automatically save outputs as local files (`gth_<timestamp>_<COMMAND>.<ext>`) instead of dumping raw base64 to the terminal.
  - Supports common image, audio, video, and PDF MIME types with automatic extension mapping.
  - Controlled via the `writeBinaryOutputsToFile` configuration option (defaults to `true`). Set to `false` to disable.
  - Works in both streaming and non-streaming modes.

## Improvements
- A2UI surface tool description now includes inline schema examples, improving model adherence to the A2UI protocol.
- Added `CONTRIBUTING.md` and `CODE_OF_CONDUCT.md`.

## Maintenance
- Routine dependency updates.
- CI workflow permissions tightened; review workflow re-enabled on PRs.
