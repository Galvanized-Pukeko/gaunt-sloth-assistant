# v1.5.5 Change Requirements Discovery

## New Features
- **Change requirements discovery:** Running `gth pr` with no arguments now reproduces the same review CI runs through the slim `@gaunt-sloth/review` package, locally and without manual bookkeeping. The command detects the current branch's open PR via `gh`, fetches the diff and PR metadata deterministically, and extracts requirements from the issue linked in the PR description — a Jira ticket (Atlassian browse URL or bare key) or a GitHub issue (URL or number). No API keys to manage and no PR or requirement numbers to type by hand.
  - When the diff and requirements are both resolved deterministically, the MCP discovery agent is skipped entirely.
  - The Jira REST fast path falls through to the MCP discovery agent when no Jira REST API key is configured, so OAuth-only setups work without extra credentials.
- **Per-command tool allow-lists:** `commands.pr.discovery.allowedTools` constrains the discovery agent to a minimal tool set, and an `allowedTools` array (top-level or per command) restricts the review agent. An empty array disables all tools and skips contacting MCP/A2A servers — including the OAuth round-trip — so the review agent can run with no tools at all.

## Improvements
- **Agent authentication failures are surfaced.** The review/PR agent now prints the underlying error instead of a generic "Failed to run review with agent." line. Google Vertex AI auth failures are detected for both HTTP 401/unauthorized responses and gcloud ADC reauthentication errors (`invalid_grant` / `invalid_rapt`), with guidance to run `gcloud auth login` and `gcloud auth application-default login`.
- Consistent error handling across the `gh` tools, with case-normalized issue URLs and clearer feedback when a PR or issue cannot be resolved.
- A second `Ctrl+C` during a raw-mode interrupt now force-exits; the Escape keypress listener is unregistered when stream creation fails.
- Output spacing around discovery metadata and review agent output tidied up.
