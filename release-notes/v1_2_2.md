# v1.2.2 Improved `gsloth init`

## New Features
- **Interactive Provider Selection:** `gsloth init` without arguments now auto-detects API keys in environment variables (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, etc.), lists available providers, and prompts the user to select one. The `[type]` argument is now optional.
- **Folder Configuration Mode:** `gsloth init` now creates a `.gsloth/` directory by default, placing config in `.gsloth/.gsloth-settings/`. Existing project-root configs continue to work.
- **`noDefaultPrompts` Option:** When set to `true` in config, skips fallback to bundled `.gsloth.*.md` prompt files — only user-provided prompts are used. Applies to all prompt files: backstory, system, chat, code, guidelines, review.

## Improvements
- **Guidelines Separation:** `.gsloth.guidelines.md` is now blank by default since it ships as the user fallback. Internal development guidelines consolidated into `AGENTS.md`.
